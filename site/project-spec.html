For a project to be a kit, it needs to do 2 things:

  • Have a KitSpec file in the root of the project; and
  • Conform to the expected Kit project structure, so that it can be packaged.

KitSpec format

KitSpec is a JSON formatted file with the follow structure:

{
      "name": "functionalkit"
    , "version": "1.0"
    , "dependencies" : [
      { "name": "blocks", "version": "0.9"}
    ]
}

Every kit, and every project that wants to depend on kits, must have a KitSpec.

Package Structure

Kit packages are expected to follow a specific folder structure:

/
    src/ 
    test/
    KitSpec
  • All headers and source files must be in the src folder. This folder is also a header search path for dependent projects.
  • Test files go in test. However, Kit currently does nothing with these files.
  • KitSpec must be in the root of the project.
